# Item Types

> Reference documentation for the 7 system item types in DevStash.
> Generated by `/research item-types-research`.

Every stashed thing is an **Item**, and every item has an **ItemType**. DevStash
ships with 7 fixed **system types** (`isSystem: true`, no owner). Custom,
user-owned types are planned (schema supports `ItemType.userId`) but not yet
seeded.

> **Note on sources:** the research prompt listed `src/lib/constants.tsx`, which
> does not exist. There is no single constants file — the type definitions live
> across `prisma/seed.ts` (canonical name/icon/color), the lucide icon map in
> `src/components/dashboard/ItemTypeIcon.tsx`, the display/routing logic in
> `src/lib/db/items.ts` (`getSidebarItemTypes`), and the mock UI shape in
> `src/lib/mock-data.ts`. Values below are reconciled from all of these plus
> `context/project-overview.md`.

`ContentType` enum (`prisma/schema.prisma`) has three values — **`TEXT`**,
**`FILE`**, **`URL`** — that back the three "content kinds" above.

## Per-Type Detail

### 1. Snippet

- **Stored name:** `snippet` (singular, lowercase) · **Label:** Snippets
- **Icon:** `Code` · **Color:** `#3b82f6` (blue)
- **Content kind:** text → `ContentType.TEXT`
- **Route:** `/items/snippets`
- **Purpose:** Reusable code — hooks, utilities, patterns, boilerplate.
- **Key fields:** `content` (the code), `language` (e.g. `typescript`,
  `dockerfile`) for syntax highlighting.

### 2. Prompt

- **Stored name:** `prompt` · **Label:** Prompts
- **Icon:** `Sparkles` · **Color:** `#8b5cf6` (purple)
- **Content kind:** text → `ContentType.TEXT`
- **Route:** `/items/prompts`
- **Purpose:** Reusable AI prompts, system messages, and workflow templates.
- **Key fields:** `content` (the prompt text); `language` typically unused.

### 3. Note

- **Stored name:** `note` · **Label:** Notes
- **Icon:** `StickyNote` · **Color:** `#fde047` (yellow)
- **Content kind:** text → `ContentType.TEXT`
- **Route:** `/items/notes`
- **Purpose:** Freeform notes, explanations, study material (markdown editor).
- **Key fields:** `content` (the note body).

### 4. Command

- **Stored name:** `command` · **Label:** Commands
- **Icon:** `Terminal` · **Color:** `#f97316` (orange)
- **Content kind:** text → `ContentType.TEXT`
- **Route:** `/items/commands`
- **Purpose:** Shell one-liners and CLI commands worth remembering.
- **Key fields:** `content` (the command), `language` (usually `bash`).

### 5. Link

- **Stored name:** `link` · **Label:** Links
- **Icon:** `Link` · **Color:** `#10b981` (emerald)
- **Content kind:** url → `ContentType.URL`
- **Route:** `/items/links`
- **Purpose:** Bookmarked docs, references, and resources.
- **Key fields:** `url` (the link). `content`/`language` unused.

### 6. File _(Pro)_

- **Stored name:** `file` · **Label:** Files
- **Icon:** `File` · **Color:** `#6b7280` (gray)
- **Content kind:** file → `ContentType.FILE`
- **Route:** `/items/files`
- **Purpose:** Uploaded files (context files, specs, docs). **Pro-gated.**
- **Key fields:** `fileUrl` (Cloudflare R2), `fileName`, `fileSize` (bytes).

### 7. Image _(Pro)_

- **Stored name:** `image` · **Label:** Images
- **Icon:** `Image` · **Color:** `#ec4899` (pink)
- **Content kind:** file → `ContentType.FILE`
- **Route:** `/items/images`
- **Purpose:** Uploaded images (diagrams, screenshots, UI references). **Pro-gated.**
- **Key fields:** `fileUrl` (Cloudflare R2), `fileName`, `fileSize` (bytes).

## Content-Kind Classification: text vs. file vs. URL

The 7 types collapse into **3 storage shapes**, driven by the `ContentType`
enum on `Item`:

| Content kind | `ContentType` | Types | Primary storage field |
| --- | --- | --- | --- |
| **text** | `TEXT` | snippet, prompt, note, command | `content` (+ optional `language`) |
| **url** | `URL` | link | `url` |
| **file** | `FILE` | file, image | `fileUrl` (+ `fileName`, `fileSize`) — Pro only |

`ContentType` defaults to `TEXT` on `Item`. In `prisma/seed.ts`, the shape is
derived per item: a `url` present → `URL`, otherwise `TEXT` (`content` +
`language`). File-based items are not seeded (Pro/R2 not wired up yet).

**Pro gating maps cleanly onto the `file` kind:** the two Pro types (file,
image) are exactly the two `FILE`-kind types. `ItemType` has **no `isPro`
column** — it's derived from the type name via
`PRO_TYPE_NAMES = { "file", "image" }` in `src/lib/db/items.ts`.

## Shared Properties (all types)

Every item, regardless of type, carries these `Item` fields:

- `id`, `title`, `description`
- `contentType` (`TEXT` | `FILE` | `URL`)
- `isFavorite`, `isPinned` (both default `false`)
- `userId` (owner; cascade delete), `itemTypeId` (its type)
- `tags` (implicit many-to-many, globally-unique tag names)
- `collections` (many-to-many via `ItemCollection`; an item can be in several)
- `createdAt`, `updatedAt`

Every **ItemType** carries: `id`, `name`, `icon` (lucide name string), `color`
(hex), `isSystem`, optional `userId` (null for system types).

## Display Differences

Where the types diverge in the UI:

- **Icon & color are data-driven.** The `icon` string maps to a lucide
  component via `ICONS` in `src/components/dashboard/ItemTypeIcon.tsx`
  (`Code`, `Sparkles`, `StickyNote`, `Terminal`, `Link`, `File`, `Image`;
  fallback `Code`). `color` is applied as an **inline style** (not a Tailwind
  class) since it's per-type data. Used consistently for sidebar link icons,
  card left borders, and collection-card tints/badges.

- **Card preview text depends on content kind** (`itemPreview` in
  `src/lib/db/items.ts`):
  - `URL` → shows `url` (falls back to `description`)
  - `FILE` → shows `fileName` (falls back to `description`)
  - `TEXT` → shows `content` (falls back to `description`)

- **Labels & routes are pluralized from the singular stored name**
  (`getSidebarItemTypes`): `snippet` → label "Snippets", slug `snippets`,
  route `/items/snippets`.

- **Sidebar order is fixed** by `TYPE_ORDER`:
  snippet → prompt → note → command → link → file → image
  (matches the overview's item-types table, not DB insertion order).

- **Pro badge:** file & image render a ShadCN `secondary` "PRO" badge in the
  sidebar; every type (including Pro) shows its live item count in the
  trailing slot.

- **Syntax highlighting** applies only to text types that set `language`
  (snippets, commands); prompts/notes typically omit it.

## Seed Coverage

`prisma/seed.ts` seeds all 7 types but only creates items for 4 of them:

| Type | Seeded items |
| --- | --- |
| snippet | 5 (React hooks, context, `cn()`, Dockerfile) |
| command | 5 (git reset, docker prune, kill port, npm list, migrate+start) |
| link | 6 (Docker, GitHub Actions, Tailwind, shadcn, Radix, Lucide) |
| prompt | 3 (code reviewer, doc generator, refactor assistant) |
| note | 0 |
| file | 0 (Pro / R2 not wired) |
| image | 0 (Pro / R2 not wired) |

A collection's `defaultTypeId` is computed as the type it holds most.
